Post-fix update:

- The strict content-dedup clean R33 dataset has `56,351` rows, not the full
  `275,396` source rows, because verifier-equivalent token/text content dedup
  drops duplicate-content components rather than assigning whole components to
  one split.
- Packed THD AR critic smokes at larger microbatch sizes failed the live
  reward-path vs training-path equivalence check with roughly `14-22%` max
  MSE-ratio divergence, depending on whether copied Nemotron-H remote code had
  the Mamba `seq_idx` boundary patch. Root cause: packed `[1, sum(T)]` Nano
  critic training was not equivalent to the padded-mask reward/eval path.
- The clean batching path as of `2026-06-12T19:44Z` is padded masked AR critic
  training with explicit per-sample last-token value indexing. Proof run
  `nano-ar-r33-dedup-throughput-smoke-lr2e5-warmup2-gb192-mb16-4step-padded`
  passed `[NLA STEP0 CHECK] reward/train MSE ratio: mean=1.0000 max|r-1|=0.0000 n=32`,
  completed four optimizer steps, and ran steady-state at about `24-25s/step`
  for `gb192/mb16` on two H200s. The temporary 72G checkpoint was deleted after
  preserving logs/run plan.
- Next clean R33 AR quality smoke should use the padded critic path with
  batched microbatches, not the old `mb1` fallback and not packed THD.
