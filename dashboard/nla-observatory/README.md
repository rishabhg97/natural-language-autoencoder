# NLA Offline Observatory

A local-first, static evidence viewer for the completed Nano30B R33 NLA
evidence bundle. Four stations — **CHANNEL** (what survives language),
**TRACE** (what changes across tokens, with the poetry planning lens),
**BENCH** (precomputed counterfactuals with compulsory control lanes), and
**AUDIT** (what is safe to claim) — over one verified, precomputed bundle.

The browser never loads model checkpoints, never calls an inference server,
and never regenerates evidence. Every displayed number is read from static
shards generated by a fail-closed builder, and headline metrics link back to
their provenance records.

## Published site

The repository publishes this application at
<https://rishabhg97.github.io/natural-language-autoencoder/> using
`.github/workflows/pages.yml`. Vite emits relative asset URLs, so the same
build works under the GitHub Pages project path and on a local static server.
The checkpoint-free `public/data` bundle is intentionally committed; Pages
does not need access to RunAI, S3, model weights, or the private raw artifacts.

Before publishing a committed bundle, run:

```bash
npm run data:verify:published
npm test
npm run build
```

## Prerequisites

- Node 20+ and [uv](https://docs.astral.sh/uv/). Python and data-builder
  dependencies are pinned by `pyproject.toml` and `uv.lock`.
- The compact evidence copied locally (gitignored):
  - `artifacts/observatory/r33_offline_v1/bundle` (core bundle, minus
    `vectors/all.f16.bin` and `geometry_basis.npz`, which are excluded by design)
  - `artifacts/observatory/r33_poetry_planning_v1` (poetry planning pack,
    minus extraction shards and `reconstructions.npz`)
  - `artifacts/runai_eval/r33_u342_research_abc_v1_20260722/eval384_matched_v122`
    (matched 122-family SFT/RL round-trip reports; no checkpoints)

## Build and run

```bash
cd dashboard/nla-observatory
npm install
uv sync --frozen

# 1. Generate browser shards from the raw evidence (fail-closed):
#    verifies bundle manifest hashes, matched SFT/RL protocol + row identity,
#    poetry report/config hashes, row counts, referential joins, control-group
#    completeness, and re-derives the
#    family-clustered bootstrap aggregates bit-exactly before emitting.
npm run data:build

# 2. Verify the generated data (hashes, joins, counts, byte determinism):
npm run data:verify

# 3. Start the dashboard:
npm run dev          # http://localhost:5199
```

## Tests

```bash
npm test             # Vitest: unit + component + axe accessibility checks
npx playwright install chromium   # once
npm run e2e          # Playwright: desktop/tablet/mobile screenshots + checks
npm run typecheck
```

Unit and component tests run against a tiny committed synthetic fixture under
`tests/fixtures/` — never against real evidence. Playwright runs against the
real generated data and writes screenshots to `screenshots/<viewport>/`.

## Data layout

```
public/data/                  # generated, gitignored
  manifest.json               # dashboard manifest: source + generated hashes
  rows.json                   # 50-row qualified validation panel
  channel.json                # matched RL result, waterfall, ladder, occlusion, …
  rewrites.json               # paraphrase/corruption court cells (both critics)
  trace.json                  # 10-document fresh-forward film set
  poetry.json                 # poetry planning lens (8 cases, controls, edits)
  bench/index.json            # intervention coverage + control groups
  bench/row-<row_id>.json     # per-row behavior detail (24 rows)
  audit.json                  # claim ledger, court docket, provenance
  reports/*.json              # byte-identical upstream report copies
```

The global PCA table is intentionally not shipped. BENCH embeds only the
per-row, validation-fitted display coordinates that its reconstruction compass
actually consumes.

`scripts/build_static_data.py` is the only layer that reads raw evidence.
`scripts/verify_static_data.py` re-verifies everything, including that a
rebuild reproduces identical bytes.

## Evidence semantics honored by the UI

- The stored-snapshot channel claim is **qualified**; fresh-forward trace
  views and validation-fitted summaries are **exploratory**; failed steering
  and weak lifts are shown as **negative** results, never hidden.
- The online-RL headline is a **matched validation** result: 0.309 SFT to
  0.224 RL round-trip directional error (27.4% lower) on the same 122 held-out
  content families and 384-token generation protocol. It is a joint AV+AR
  effect, not actor attribution or a sealed test-set claim.
- Pipeline `passed` flags mean phase completeness, not scientific success.
- Control lanes (identity, paraphrase-placebo, random-edit; real-vs-shuffled)
  travel with their cells and cannot be dismissed.
- Descriptions are learned encodings of stored activations — the UI never
  calls them the model's thoughts.
- TRACE binds sampled trajectory tokens to released source text only when the
  tokenizer ids match exactly. The current bundle has exact source offsets for
  360/400 positions; all 40 positions from one unaligned document show an
  explicit unavailable state instead of a guessed source highlight.
- Document and poetry readers preserve the visual chain from original text to
  highlighted token, activation, NLA verbalization, controls, and observed
  model output. Sampled positions are selected directly inside the full source
  text; previous/next buttons are the only secondary navigation.
