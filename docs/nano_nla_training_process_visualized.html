Key mechanics

  • Injection hook is registered only for SFT/policy actors train_actor.py:503-506. It clones the embedding output and overwrites rows where the token is the injection marker and the left/right neighbors match — site count must equal vector count or it aborts injection.py:79-92.
  • Scale: the raw activation is L2-normalized to injection_scale (REQUIRED for AV; 75 in the layer-probe config) train_actor.py:441-449, 600.
  • Loss: Miles stock sft_loss; the actor's _compute_log_prob no-ops under SFT train_actor.py:661-662.
  • Checkpoint: DCP only. To reload: --hf-checkpoint = base model, --load = iter dir train_actor.py:871-877.

Key mechanics

  • No hook — the critic consumes explanation tokens directly; the hook is AV-only train_actor.py:503-506.
  • Critic model: backbone truncated to K+1 blocks; lm_head & final layernorm → Identity; bias-free Linear(d,d) value head models.py:137-274.
  • Extraction: the template ends in a fixed suffix (e.g. </text> <summary>), so the last real token is the extraction point — no scanning.
  • Loss / scale: direction-only MSE; mse_scale defaults to sqrt(d_model) config.py:182-183. Logs fve_nrm, pred/gold norms.

FSDP2 / sharding

Current runs are pure DP/FSDP over 2 GPUs, cp_size=1, tp_size=1 (asserted train_actor.py:373-377; --actor-num-gpus-per-node defaults to 2 nano_av_runner.py:252-254).

Ray actor group (1 node) · mesh = parallel_state.dp_mesh · apply_fsdp2 / fully_shard over transformer blocks + top-level model

model shard 0 — bf16 params (compute)
fp32 master weights — shard 0
Adam optimizer state — shard 0 (~½ of ~118 GiB)

GPU 1 — Ray actor (rank 1)

model shard 1 — bf16 params (compute)
fp32 master weights — shard 1
Adam optimizer state — shard 1 (~½ of ~118 GiB)
⇄ NCCL: forward all-gather params per block · backward reduce-scatter grads (fp32 reduce) · torch.cuda.synchronize() bounds the overlap peak train_actor.py:714-726
bf16 params (compute) fp32 master Adam state (sharded)
  • Ray actor per GPU: one NLAFSDPActor (a FSDPTrainRayActor) per GPU; collectives like critic_fwd and the cross-rank MIN truncation are "all ranks must call" train_actor.py:729-756, 156-180.
  • bf16 / fp32: MixedPrecision is compute-only; model stored fp32, cast fp32→bf16 on save for smaller checkpoints train_actor.py:843-856.
  • Optimizer state on sharded params: a 2-rank Adam checkpoint is ~177 GiB (~59 GiB model + ~118 GiB optimizer); NLA_KEEP_LOCAL=1 keeps only the latest exact-resume checkpoint.
  • Stability fixes: non-reentrant gradient checkpointing (use_reentrant=False) train_actor.py:492-496; DCP object/metadata collectives routed through a Gloo PG while training stays on NCCL (Miles patch 0006).
