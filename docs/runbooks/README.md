# Runbooks

Runbooks are operational procedures for repeating or maintaining experiments.
They should avoid scientific interpretation and focus on steps, checks, and
failure recovery.

| File | Purpose |
|---|---|
| `runai_s3_sync.md` | Keep Mac, RunAI, S3, and GitHub source states aligned |
| `runtime_monitoring.md` | Capture GPU/process/system telemetry during training |
| `../cluster_runbook.md` | Existing cluster command reference |
| `../runai_av_100k_repeatability.md` | Existing AV 100k repeatability workflow |
